# Release Slice Contract

## Purpose

This file is the working nucleus for the current release effort.
It exists to prevent drift.

Every future change to the release slice must be checked against this file before implementation.

If a change improves a local detail but weakens the whole experience, it must be rejected.

## Current Slice

The only slice currently in scope is:

- Splash
- Home
- Soglia
- Giardino
- first true Bach reward moment

No other slice may be expanded until this one feels coherent.

## Core Goal

The game must stop feeling like a sequence of text responses and start feeling like a ritual journey through emotionally distinct chambers.

The release slice must create:

- desire to continue
- atmosphere with pressure and relief
- a sense of earned revelation
- a clear difference between ordinary response and meaningful event

## Non-Negotiable Rules

### 1. Bach is revelation, never furniture

Bach must never function as continuous decorative background.

Bach may play only when something has been earned:

- puzzle solved
- simulacrum found
- sector breakthrough
- finale threshold
- true revelation

When Bach appears:

- it must be brief
- it must not overlap with another Bach cue
- it must return to ambience after the cue finishes

### 2. The world must sound present even when music is absent

Default bed = material atmosphere, not melodic bed.

Allowed default materials:

- air
- water
- metal
- paper
- resonance
- faint drone

Forbidden default behavior:

- full Bach loop as room bed
- reward cue repeating like UI confirmation sound
- random loudness jumps without narrative reason

### 3. Not every response deserves the same treatment

The game must distinguish clearly between:

- minor response
- solved puzzle
- simulacrum found
- sector transition
- major revelation
- finale threshold

If two response types feel the same, the hierarchy is failing.

## Response Hierarchy

### Minor response

- text appears normally
- no Bach
- no big visual punctuation

### Solved puzzle

- short pause before text
- light reward cue only
- should feel like progress, not climax

### Simulacrum found

- screen confirmation
- stronger reward cue
- longer pause than puzzle solved
- should feel like a memorable event

### Sector transition

- visible shift
- black fade
- new atmosphere must be felt quickly

### Major revelation

- silence before text
- text reveal must slow down and gain weight
- should feel irreversible

### Finale threshold

- normal flow breaks
- system feels altered
- the player must sense that the rules have changed

## Text Rhythm Rules

The typewriter is not the universal default anymore.

### Typewriter

Use for:

- standard room flow
- ordinary commands

### Slow reveal

Use for:

- first room descriptions when the room must land with weight
- entrances that establish a chamber

### Word-by-word

Use for:

- revelations
- simulacra
- spiritually weighted text

### Instant

Use for:

- Demiurge interruptions
- responses that should feel like intrusion, not narration

## Splash Contract

Splash must not feel like a random beautiful screen.
It must establish the law of the game.

Required:

- no random Bach furniture
- stillness
- anticipation
- explicit sense of entering a world, not opening a menu

Failure condition:

- if the splash feels interchangeable with another literary game, it is wrong

## Home Contract

Home must not feel like an app dashboard.
It must feel like a threshold between reality and descent.

Required:

- prestige
- clarity
- ritual framing

Failure condition:

- if the player feels they are browsing options instead of preparing to descend

## Soglia Contract

Soglia is not a menu hub.
Soglia is suspension.

Required:

- faint resonance
- emptiness
- structural calm
- clear sense of centrality

When returning after meaningful progress:

- the player must feel the Archive has changed slightly

Failure condition:

- if Soglia feels like neutral navigation infrastructure

## Giardino Contract

The Garden must have a readable emotional arc:

- arrival
- curiosity
- friction
- relinquishment
- revelation
- peace

The player must not experience the Garden as a flat chain of puzzles.

Atarassia must feel:

- quieter than normal success
- heavier than normal success
- spiritually cleaner than the rest of the sector

Failure condition:

- if Atarassia feels like just another item acquisition

## Acceptance Criteria For The Slice

The slice is only considered successful if all these are true:

- Splash contains no decorative Bach bed
- Soglia feels distinct from ordinary rooms
- Giardino does not feel rhythmically flat
- Atarassia feels different from a standard reward
- Bach is scarce enough to be desired
- the player can sense progression emotionally, not only logically

## What Must Not Happen

- no new sector work before this slice is coherent
- no new content added just to compensate for weak rhythm
- no feature creep outside rhythm, atmosphere, and emotional differentiation
- no local polish that contradicts the global contract

## Working Rule For Future Sessions

Before making changes, the agent must answer:

1. Which rule in this contract is being improved?
2. Which rule could be broken by this change?
3. Does this make the slice more distinct, or only more busy?

If those answers are unclear, do not patch the game yet.
